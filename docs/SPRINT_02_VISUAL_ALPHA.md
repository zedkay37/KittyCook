# Sprint 02: Visual Alpha Massive Upgrade

Goal: make the prototype immediately read as KittyCook from a screenshot without
changing the gameplay rules from `v0.1-alpha`.

## Delivered Direction

- Procedural Three.js model factories for characters, stations, items, kitchen
  set dressing, and VFX.
- Cartoon cat chefs with ears, hats, tails, paws, eyes, player rings, idle/walk
  motion, boost squash, and slip wobble.
- Distinct station silhouettes for ingredient crates, plate stack, cutting,
  cooking, assembly, service, and trash.
- Distinct item silhouettes for fish, bread, herb, plates, dishes, cooked and
  burned states.
- Decorated `Cushion Counter` kitchen set with tile floor, counters, cushion rug,
  paw prints, service glow, and KittyCook sign.
- Event-driven VFX for meow, score, cutting, cooking, burning, and slipping.
- HUD polish for party readability with compact controls hint.

## Acceptance Checklist

- A screenshot should read as a cartoon cat cooking party game.
- Every station should be identifiable by silhouette and color.
- Fish, bread, herb, plate, dish, cooked, and burned states should be distinct.
- VFX should make actions clearer without hiding objects or stations.
- Gameplay from `v0.1-alpha` should still work.
- Build must pass with `npm.cmd run build`.
- Browser console should show no runtime errors on load.
